Security and Privacy (امنیت و حریم خصوصی)

این سند، استراتژی‌ها، رویه‌ها و کنترل‌های امنیتی مورد نیاز برای محافظت از داده‌ها، مدل‌های هوش مصنوعی و زیرساخت پلتفرم Vira AI را تشریح می‌کند.

۱. اصول امنیتی (Security Principles)

امنیت در Vira AI بر اساس رویکرد «امنیت از پایه» (Security by Design) بنا شده و شامل سه اصل کلیدی است:

اصل کمترین امتیاز (Principle of Least Privilege - PoLP): هر کاربر یا سرویس تنها به حداقل منابع لازم برای انجام وظیفه‌اش دسترسی دارد.

دفاع در عمق (Defense in Depth): استفاده از چندین لایه امنیتی مستقل برای جلوگیری از نقض امنیتی.

بی‌اعتمادی هرگز (Never Trust, Always Verify): پیاده‌سازی مدل امنیت Zero Trust در سراسر شبکه داخلی و خارجی.

۲. امنیت دسترسی و احراز هویت (Access and Authentication Security)

۲.۱. احراز هویت (Authentication)

SSO/OAuth2: استفاده از Single Sign-On و پروتکل OAuth2 برای احراز هویت کاربران.

MFA (Multi-Factor Authentication): اجباری شدن احراز هویت چند عاملی برای تمام حساب‌های مدیریتی و حساس.

۲.۲. مجوزدهی (Authorization)

RBAC (Role-Based Access Control): پیاده‌سازی کنترل دسترسی مبتنی بر نقش برای محدود کردن عملیات کاربران (مثلاً نقش‌های Viewers، Developers، و Administrators).

Service-to-Service Authorization: استفاده از توکن‌های کوتاه‌مدت (مانند JWT) برای احراز هویت و مجوزدهی بین میکروسرویس‌های داخلی.

۳. رمزنگاری و محافظت از داده‌ها (Encryption and Data Protection)

۳.۱. رمزنگاری در حال انتقال (Encryption in Transit)

تمام ارتباطات بین کاربر و پلتفرم، و همچنین بین میکروسرویس‌های داخلی (در صورت لزوم)، باید با استفاده از TLS/SSL (حداقل نسخه ۱.۲) رمزگذاری شوند.

۳.۲. رمزنگاری در حالت سکون (Encryption at Rest)

داده‌های حساس (Sensitive Data): تمام داده‌های کاربر، پایگاه‌های داده و فایل‌های ذخیره‌سازی شیء (Object Storage) باید با استفاده از استاندارد AES-256 رمزنگاری شوند.

کلیدهای رمزنگاری: مدیریت کلیدهای رمزنگاری باید توسط یک سرویس مدیریت کلید امن (مانند HashiCorp Vault یا سرویس‌های KMS ارائه‌دهندگان ابری) انجام شود.

۴. امنیت زیرساخت و شبکه (Infrastructure and Network Security)

تقسیم‌بندی شبکه (Network Segmentation): استفاده از Network Policies در Kubernetes برای اطمینان از اینکه هر میکروسرویس تنها با سرویس‌های مورد نیاز خود در ارتباط است.

مدیریت آسیب‌پذیری (Vulnerability Management): اسکن منظم Imageهای Docker و Dependencyهای نرم‌افزاری برای شناسایی و رفع آسیب‌پذیری‌های امنیتی (CVEs).

حفاظت از Endpoint: استفاده از Web Application Firewall (WAF) برای فیلتر کردن ترافیک مخرب در لایه API Gateway.

۵. امنیت خاص هوش مصنوعی (AI/ML Specific Security)

مدل‌های هوش مصنوعی نیازمند اقدامات امنیتی اضافی هستند تا از حملات خاص این حوزه محافظت شوند:

۵.۱. حفاظت از مدل (Model Integrity)

Model Poisoning Prevention: اعتبارسنجی دقیق داده‌های آموزشی جدید برای جلوگیری از آلودگی مدل توسط مهاجمان.

Model Intellectual Property Protection: رمزنگاری و امن‌سازی Model Artifacts در Object Storage و محدود کردن دسترسی به آن‌ها.

۵.۲. حفاظت در برابر حملات استنتاج (Inference Attack Prevention)

Model Extraction Attacks: محدود کردن نرخ (Rate Limiting) و مانیتورینگ دقیق APIهای استنتاج برای تشخیص تلاش‌ها برای استخراج معماری یا پارامترهای مدل.

Adversarial Examples: پیاده‌سازی تکنیک‌های دفاعی (مانند Adversarial Training) برای کاهش حساسیت مدل به ورودی‌های دستکاری‌شده که به منظور فریب مدل طراحی شده‌اند.

۶. حریم خصوصی داده (Data Privacy)

حداقل‌سازی داده‌ها (Data Minimization): فقط داده‌هایی را جمع‌آوری و ذخیره کنید که برای ارائه خدمات ضروری هستند.

ناشناس‌سازی (Anonymization/Pseudonymization): استفاده از تکنیک‌های ناشناس‌سازی برای داده‌های حساس در هنگام آموزش و استفاده مجدد.

حذف داده‌ها: پیاده‌سازی سیاست‌های روشن و کارآمد برای حذف ایمن و کامل داده‌های کاربران پس از درخواست یا انقضای دوره نگهداری.
